Every drained wallet you've read about on Twitter was a control problem, not a model problem.

from ghostgate import (
    GatedWallet, MockSigner, MockRPC, policies,
    TxDenied, WalletFrozen,
)

UNISWAP = "0x7a250d5630B4cF539739dF2C5dAcb4c659F2488D"
SCAM    = "0xDeadBeef00000000000000000000000000000000"

wallet = GatedWallet(
    signer=MockSigner(),
    rpc=MockRPC(),
    policies=[
        policies.contract_denylist({SCAM}),
        policies.contract_allowlist({UNISWAP}),
        policies.max_value_per_tx(max_wei=10**17),        # 0.1 ETH hard cap
        policies.rate_limit(max_sends=5, window_seconds=60),
        policies.spend_cap(max_wei=10**18, window_seconds=3600),
    ],
)

# Normal bot behavior — fine.
wallet.send(UNISWAP, value_wei=10**16)

# Bot gets prompt-injected and tries to drain.
try:
    wallet.send(SCAM, value_wei=10**18)
except WalletFrozen as e:
    print("stopped:", e)
    # -> stopped: attempted send to denylisted address 0xDeadBeef...

# Wallet is now locked. Every subsequent send raises WalletFrozen
# until a human calls wallet.unfreeze().
APPROVE

All rules passed. Sign and broadcast.

DENY

This transaction blocked. Wallet stays usable.

FREEZE

All future sends blocked until a human unfreezes.

What GhostGate is NOT.

×
Not a wallet. It wraps one. Your signer, your keys, your problem.
×
Not a bot. It gates one. You bring the strategy, Gate enforces the rails.
×
Not an AI layer. Policies are deterministic on purpose. No model in the critical path, ever.
×
Not a dashboard product. The audit trail is a stream, not a pretty screen. Pipe it anywhere.

Gate doesn't decide what to do.

It decides what is allowed to happen.
